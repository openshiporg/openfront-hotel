# RUN_LOG.md

Use this file as a lightweight overnight journal.

## How To Log
- Append short dated entries.
- Focus on what was actually changed, what parity gap or user story it addressed, and what remains.
- Keep entries concise and useful for a fresh morning review agent.
- Do not fabricate progress.

## Entry Template
### YYYY-MM-DD HH:MM
- Worked on:
- Why it mattered:
- Main files touched:
- Remaining gap / next step:
- Confidence: high / medium / low
